d) This should always generate the same score as GlobalAlignerPlus. The only difference between the two programs is that AffineGlobalAlignerPlus has to account for the gap opening penalty and the existence of previous gaps. However, if you make the affine gap opening penalty zero, the two programs are essentially computing the same values for each element in the table and therefore, they should yield the same optimal alignments.

e) It is predicted that the running times of the two programs will be similar but AffineGlobalAlignerPlus is expected to take slightly longer to run. The time complexity of both algorithms is O(mn) but when running the AffineGlobalAlignerPlus program, you have to take the extra step of checking if a gap is present when making each calculation. This extra step is expected to make a very small difference but will still take longer than GlobalAlignerPlus which does not have to take in the presence of the gap during each table entry calculation. Due to this difference, the real time elapsed for GlobalAlignerPlus should be slighter less than the real time elapsed while running AffineGlobalAlignerPlus.
